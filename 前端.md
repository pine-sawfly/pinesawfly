我想实现一个类似 material-components-pyside 的桌面 UI 项目。
请先分析项目架构并给出目录结构和实现计划，确认后再开始生成代码。
技术栈：
- 使用 Python + PySide6
- 前端界面使用 Qt Quick / QML
- Python 负责启动应用、加载 QML、暴露主题管理对象
- QML 负责所有界面、控件、布局、动画和页面切换

目标：
做一个 Material Design 3 风格的桌面组件库和 Demo Gallery。

项目结构希望类似：
- app.py：创建 QGuiApplication 和 QQmlApplicationEngine，加载 Main.qml
- stylemanager.py：管理亮色/暗色主题、种子色、动态配色
- qml/App/Main.qml：主窗口
- qml/App/AppRoot.qml：应用根布局，包含左侧导航和右侧页面 Loader
- qml/App/pages/：示例页面
- qml/Core/Controls/：自定义控件
- qml/Core/Styles/Theme.qml：全局主题 singleton
- qml/md3/Core/qmldir：注册 QML 组件模块

界面要求：
- 主窗口尺寸 1280x800
- 左侧 NavigationDrawer 导航栏
- 右侧内容区域根据菜单动态加载页面
- 页面切换有淡入和位移动画
- 支持 Home、Components、Color、Typography、Settings、About 等页面
- Components 页面展示 Button、Card、Switch、Checkbox、Slider、TextField、Dialog、Tabs、FAB 等控件
- Color 页面展示当前亮色/暗色主题色板
- Settings 页面可以切换暗色模式、选择种子色，并实时更新全局主题

主题要求：
- 实现 Theme.qml singleton
- 提供 color、typography、shape、elevation、state 等 token
- 所有控件都从 Theme 读取颜色、字体、圆角、状态透明度
- Python 的 StyleManager 暴露 currentScheme、lightScheme、darkScheme、seedColor、isDarkTheme 给 QML
- 改变 seedColor 或暗色模式时，QML 界面自动刷新

控件要求：
- 每个控件独立成一个 QML 文件
- 控件 API 要清晰，例如 Button 支持 text、icon、type、enabled、clicked
- 实现 hover、pressed、disabled、focus 状态
- 实现 Ripple 点击效果
- 使用 Material Icons 字体显示图标
- 尽量遵循 Material Design 3 视觉规范

交付要求：
- 给出完整可运行项目
- 包含 requirements.txt 或 pyproject.toml
- 提供运行命令：python -m 项目名
- 代码结构清晰，方便后续扩展新控件
